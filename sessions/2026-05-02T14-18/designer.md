## 🧬 JAY CORP 디자인 시스템 가이드라인 (Design System Guide)

### 1. 비주얼 컨셉 및 무드 보드 (Visual Concept & Mood Board)
| 항목 | 정의 | 적용 이유 |
| :--- | :--- | :--- |
| **핵심 키워드** | Precision, Data-Driven, Architectural, Clarity | 복잡한 문제를 과학적이고 체계적인 방식으로 해결한다는 메시지를 시각적으로 전달. |
| **전반적 톤** | 미니멀리즘(Minimalism) 기반의 하이테크(Hi-Tech). 여백을 활용하여 정보의 가독성을 극대화하고, 강조되는 데이터에만 색상을 사용해 집중도를 높임. | 과장된 이미지나 감성적인 접근보다, 객관적 데이터를 전면에 내세워 '신뢰'를 구축합니다. |
| **핵심 메타포** | 연결(Connection), 흐름(Flow), 계층 구조(Hierarchy) | JAY CORP의 방법론이 A와 B를 끊임없이 연결하고 최적화한다는 개념을 시각적으로 표현합니다. (예: 선, 노드, 다이어그램 활용). |

### 2. 컬러 팔레트 (Color Palette)
| 역할 | 색상명/코드 | 사용 범위 및 예시 | 상징하는 가치 |
| :--- | :--- | :--- | :--- |
| **Primary Color** *(신뢰)* | `#0A1931` (딥 네이비/차콜) | 헤더 배경, 주 타이포그래피, CTA 배경. 깊은 전문성과 안정감을 부여합니다. | 신뢰성, 권위, 깊이 |
| **Secondary Color** *(배경)* | `#F5F7FA` (라이트 그레이) | 본문 섹션 배경 및 카드(Card)의 바탕색. 눈의 피로도를 낮추고 콘텐츠에 집중하게 만듭니다. | 전문성, 여백, 깔끔함 |
| **Accent Color** *(혁신)* | `#007AFF` (스틸 블루/사이언) | 데이터 하이라이트, CTA 버튼 활성화 상태, 핵심 그래프 라인, 인터랙티브 요소(Hover). 기술적이고 미래지향적인 느낌을 줍니다. | 혁신, 기술력, 에너지 |
| **Neutral Color** *(보조)* | `#6B7280` (미디엄 그레이) | 설명 문구, 부제목, 보조 아이콘 색상 등. 주색과 대비를 이루어 정보의 위계를 설정합니다. | 균형, 구조화 |

### 3. 타이포그래피 시스템 (Typography System)
*   **글꼴 선정:** Pretendard 또는 Inter (가독성과 현대적인 느낌을 동시에 제공하는 산세리프 계열 권장).
*   **폰트 스케일 (Desktop 기준):**
    *   `H1` (섹션 제목): 48px ~ 64px, Bold. (강력한 메시지 전달)
    *   `H2` (소제목/방법론 이름): 32px ~ 38px, Semi-Bold. (구조화된 정보 그룹화)
    *   `H3` (단계별 제목/핵심 포인트): 24px ~ 28px, Medium.
    *   `Body Large`: 18px, Regular. (가장 중요한 설명 문단).
    *   `Body Small`: 16px, Regular. (일반적인 본문 텍스트).

### 4. 핵심 컴포넌트 라이브러리 (Key Component Library)
| 컴포넌트 | 디자인 가이드라인 | 사용 목적 및 예시 |
| :--- | :--- | :--- |
| **CTA 버튼** | 배경: Primary Color (`#0A1931`), 텍스트: White. Hover 시 Accent Color로 그라데이션 적용. 모서리: 부드러운 곡률 (4px). | "무료 진단 요청하기", "자세히 보기". 행동을 유도하는 최우선 요소. |
| **데이터 카드** | 배경: Secondary Color (`#F5F7FA`). 경계: 얇은 Accent Color 테두리(1px solid)를 넣어 부각. 내부 구조는 아이콘 + 제목 + 짧은 설명으로 구성. | 핵심 통계치 또는 장점을 요약하여 보여줄 때 (예: "평균 비용 40% 절감"). |
| **프로세스/방법론** | 단계별(Step 1, Step 2...) 구분을 명확히 하고, 각 단계를 연결하는 시각적 화살표(`→`)나 라인 구조를 사용. 아이콘과 짧은 설명 블록을 포함하여 직관성을 높임. | JAY CORP Methodology와 같은 순서가 중요한 정보 전달에 필수. |
| **Hero 섹션** | 최소한의 텍스트 + 대형 헤드라인(H1) + 배경에 미니멀하고 추상적인 '데이터 그리드' 또는 '네트워크 노드' 패턴을 오버레이하여 혁신성을 은유적으로 표현. | 방문자가 들어오자마자 시선을 사로잡고 회사 정체성을 각인시키는 곳. |

## 💻 반응형 웹 와이어프레임 (Responsive Wireframe Blueprint)

**[적용 구조]:** Business 에이전트가 정의한 Conversion Flow를 따름.
**[반응형 고려]:** 데스크톱(Desktop) 중심 설계 후, 모바일(Mobile)에서 단일 컬럼으로의 변환을 기본 원칙으로 합니다.

### 🌐 Desktop View (1440px 기준) - 섹션별 레이아웃 구조

#### **Section 1: Hero Section (Hook & 공감)**
*   **레이아웃:** 6:4 분할 (좌측에 메시지, 우측에 시각적 요소).
*   **좌측(메시지):** H1 (최대 크기), 짧은 보조 문구, [Primary CTA] 버튼.
*   **우측(시각화):** 추상적인 배경 그래픽 (네트워크 노드/데이터 플로우) 또는 제품의 핵심 원리를 보여주는 간결한 애니메이션 모형 배치.
*   **하단:** 신뢰도 확보를 위한 로고 그리드 (파트너사 로고).

#### **Section 2: Problem Definition & Gap Analysis (문제 심화)**
*   **레이아웃:** 중앙 정렬의 데이터 카드 배열 및 통계 그래프 활용.
*   **구조:** 문제 제기(H2) $\rightarrow$ 현황 분석 섹션 ($\text{A} \rightarrow \text{B} \rightarrow \text{C}$로 이어지는 비효율적인 과정 흐름 다이어그램 배치).
*   **데이터 시각화:** 비교 차트 (막대/파이 그래프)를 사용하여 '기존 대비 손실률'을 직관적으로 보여줌.

#### **Section 3: Core Solution / JAY CORP Methodology (해결책 제시)**
*   **레이아웃:** 중앙 집중식의 프로세스 플로우 (가장 중요한 섹션).
*   **구조:** JAY CORP Methodology를 3~4개의 큰 단계(Step)로 분할. 각 Step은 아이콘 + 제목(H3) + 설명 문단으로 구성되며, 시각적 화살표(`→`)로 연결되어 흐름을 강조합니다.
*   **강조점:** 이 섹션에서는 '어떻게' 작동하는지 원리를 명확한 다이어그램으로 보여주는 것이 가장 중요함.

#### **Section 4: Differentiation & Proof Point (신뢰 구축 및 전환)**
*   **레이아웃:** 콜드 클라이언트(Before/After) 사례 기반의 그리드 구조.
*   **구조:** [사례 제목] $\rightarrow$ [문제 정의] $\rightarrow$ [JAY CORP 적용 방식] $\rightarrow$ [결과 수치 (데이터 카드 활용)]. 각 스토리가 독립적인 '성공 케이스'처럼 보이게 구성합니다.
*   **마무리:** CTA 섹션을 페이지 하단에 크게 배치하여, 다음 행동(무료 컨설팅 요청)을 다시 한번 강하게 유도합니다.

### 📱 Mobile View (375px 기준) - 반응형 변환 원칙
1.  **컬럼 축소:** 모든 다중 컬럼 구조는 순서대로 쌓이는 단일 세로 컬럼(Single Column Stack)으로 변경됩니다.
2.  **여백 조정:** 가독성을 위해 패딩(Padding)과 마진(Margin)을 충분히 확보합니다 (Desktop 대비 여백 비율 증가).
3.  **이미지 우선 배치:** 텍스트가 길어질 경우, 관련 이미지를 먼저 보여주고 그 아래에 설명을 붙이는 방식으로 정보의 흐름이 끊기지 않게 조정합니다.
